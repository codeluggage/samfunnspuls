# AGENTS.md

Shared guidance for AI coding agents working in the Next.js template for Red Cross web apps.

## Repo At A Glance

- Stack: Next.js App Router, TypeScript, `rk-designsystem`, `rk-design-tokens`.
- Primary UI rule: prefer existing `rk-designsystem` components over custom HTML for common interface elements.
- Shared skill source of truth lives in `.agents/skills`.
- Shared rule source of truth lives in `.agents/rules`.

## Non-Negotiables

- Use `rk-designsystem` components before introducing custom UI.
- Use `--ds-size-*` tokens for spacing. Do not use `--ds-spacing-*`.
- Use CSS Modules for layout and composition, not to restyle design system components.
- Prefer semantic structure: one clear `<main>`, a top-level heading, accessible names for interactive elements, and predictable heading hierarchy.
- Run `npm run check:ai` before claiming a UI change is complete.

## Skills And Rules

- Shared skills live in `.agents/skills`.
- Tool-specific copies for Claude and Cursor are generated by `npm run skills:sync`.
- If generated target files differ from their shared source, the sync script stops and shows a diff instead of overwriting them.
- Use `designsystem-proof` for non-trivial UI verification and handoff evidence.
- Use `refresh-rk-context` when pulling updated guidance from the upstream DesignSystem repo or published Storybook artifacts.

## Refresh Workflow

- `AI_DESIGN_SYSTEM_GUIDE.md` stays checked in and is part of the template baseline.
- Refreshes can update:
  - `AI_DESIGN_SYSTEM_GUIDE.md`
  - `metadata.json`
  - `.agents/context/upstream/ai-context.manifest.json`
- Refreshes must surface drift and stop for review when they imply policy or standards changes rather than a safe context update.

## Verification Defaults

- `npm run typecheck`
- `npm run lint`
- `npm run build`
- `npm run check:designsystem`
- `npm run check:a11y`
- `npm run check:agent-context`
- `npm run check:ai`

## Keep This Lean

- Keep cross-agent rules here.
- Put tool-specific formatting and adapter details in generated `.claude/rules` and `.cursor/rules`.
- Keep large reference material in the checked-in guide and upstream context files, not in this file.
