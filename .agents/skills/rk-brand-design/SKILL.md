# Røde Kors brand design

Use this skill for *design decisions* — visual treatment, layout intent, copy tone — that the upstream component API alone does not answer. It complements `AI_DESIGN_SYSTEM_GUIDE.md` (component props) and `metadata.json` (component coverage). It does not replace them.

## Read first

- `BRAND.md` — brand voice, vibe, color discipline, hero treatment, animation, hover/active behavior, iconography, casing, copy rules. The non-negotiables live here.
- `assets/` — `red-cross-logo.svg`, `logo-red-cross.svg`, `herosectionbg.png`. Copy into `public/brand/` when used in a Next.js app.
- `mocks/colors_and_type.css` — single-file `--ds-*` foundation for *throwaway HTML mocks* only. Production code uses `rk-design-tokens` + `rk-designsystem` directly.
- `mocks/website/` and `mocks/docs/` — high-fidelity HTML/JSX recreations of `rodekors.no` and the docs site. Use as visual reference when deciding hero composition, header structure, footer pattern, donate dialog.
- `mocks/preview/` — small HTML cards demonstrating each foundation (colors, type, spacing, shadows, etc.). Use when previewing a token decision.
- `tokens/theme.css` — verbatim mirror of the published `rk-design-tokens` theme. Regenerated by `refresh-rk-context`. Use this when a mock needs every named token, not just the curated subset in `mocks/colors_and_type.css`.

## When to use

- Designing a new page or section in the Next.js app.
- Choosing whether to use red, neutral, or a semantic color on a surface.
- Deciding hero block padding, backdrop treatment, CrossCorner placement.
- Translating a wireframe into rk-designsystem components.
- Producing a throwaway HTML mock for a stakeholder review.
- Reviewing existing UI for brand-conformance.

## When NOT to use

- For component props or Storybook examples — read `AI_DESIGN_SYSTEM_GUIDE.md`.
- For token names or values — read `tokens/theme.css` (mirror) or `node_modules/rk-design-tokens/design-tokens-build/theme.css`.
- For what passes static checks — run `npm run check:designsystem`.

## Brand non-negotiables (the short list)

- Red `#D52B1E` only. Never recolor the cross logo.
- One primary CTA per view. Red for *attention*, never decoration.
- Source Sans 3 only. Sentence case Bokmål Norwegian by default. **du/deg**, never *man*.
- No emoji. Aksel icons in production, Lucide CDN only in throwaway HTML mocks.
- Hover = darker shade, never lighter or scaled. No bounce, no parallax.
- 44px (`--ds-size-11`) minimum touch target. WCAG 2.2 AA across all text/background pairs.
- Hero block padding 88–120px (`--ds-size-22` to `--ds-size-30`) on desktop. Generous breathing room is part of the brand.
- CrossCorner sparingly. Never more than two per page.

## Workflow

1. **Read `BRAND.md`** before sketching. The vibe rules are short; read them.
2. **Sketch in HTML** if needed using `mocks/colors_and_type.css` or `tokens/theme.css`. These mocks are throwaways — do not check them into the app.
3. **Implement** against `rk-designsystem` in the Next.js app. Configure components through props before reaching for layout CSS. Use `--ds-size-*` for spacing.
4. **Verify** with the `designsystem-proof` skill — browser evidence, accessibility issues, console clean. The proof report should explicitly note that the brand non-negotiables above were considered.
